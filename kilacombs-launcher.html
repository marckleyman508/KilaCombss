1
Prep
Unzip Your Files
Download kilacombs-deploy.zip and unzip it. You need these files:
📄server.jsrequired
📄package.jsonrequired
📄render.yamlrequired
📁public/shop.htmlrequired
📄Procfile · railway.toml · .gitignorealso include
⚠️ public/shop.html must be inside a folder named public — not shop.html by itself at the root.
2
GitHub
Create a GitHub Repo
Render pulls your code from GitHub. Takes 2 minutes to set up.
Open GitHub → New Repo ↗
On the new repo page:

Repository name: kilacombs
Visibility: Public (free)
Everything else: leave default

Click Create repository
3
GitHub
Upload All Files
On your empty repo page, upload the contents of the kc-final folder.
1. On your repo page click "uploading an existing file"
2. Open the unzipped kc-final folder
3. Select ALL files + the public folder → drag into GitHub
4. Scroll down → click Commit changes
⚠️ Upload the contents of kc-final — not the folder itself. After uploading, server.js should appear at the root of your repo.
4
Render
Connect & Configure
Create your free Render account and connect your repo with these exact settings.
Open Render → New Web Service ↗
1. Sign up at render.com with GitHub
2. Click New + → Web Service
3. Click Connect next to your kilacombs repo
4. Use these exact settings:
Namekilacombs
RuntimeNode
Build Commandnpm install
Start Commandnode server.js
Instance TypeFree
5
Launch
Deploy & Verify
Click Create Web Service and watch the logs. Takes 60–90 seconds.
✅ Success looks like:
Running 'node server.js'
KilaCombs Full-Service API 🍯 NYC
http://localhost:10000
❌ If you see SyntaxError: Unexpected end of input — a file got cut off. Click server.js in your GitHub repo and check it has 504 lines. If fewer, delete and re-upload it.
Test your live URL once it's up:

Site: https://YOUR-APP.onrender.com
API check: https://YOUR-APP.onrender.com/api/health
Should return: {"status":"ok"}
Deploy on Railway
$5 free credit · Always on · ~5 minutes
0 of 4 done
1
GitHub
Upload Files to GitHub
Railway also deploys from GitHub. Create a repo and upload your kc-final files.
Create GitHub Repo ↗
1. Name repo kilacombs → Create
2. Click "uploading an existing file"
3. Upload contents of kc-final folder
4. Commit changes
2
Railway
Deploy from GitHub
Two clicks to get it running on Railway.
Open Railway → New Project ↗
1. Sign up at railway.app with GitHub
2. Click Deploy from GitHub repo
3. Select your kilacombs repo
4. Railway detects Node automatically → click Deploy Now
3
Railway
Generate Your Domain
Railway doesn't auto-assign a public URL — you generate it in Settings.
1. Click your service in Railway dashboard
2. Go to Settings tab
3. Scroll to Networking → Public Networking
4. Click Generate Domain
5. Your URL: kilacombs-xxx.up.railway.app
4
Launch
Verify It's Working
Hit your Railway URL and confirm site + API are both live.
Site: https://YOUR-APP.up.railway.app
API: https://YOUR-APP.up.railway.app/api/health → {"status":"ok"}
✅ Site · API · Excel download at /api/excel/download · Admin at /api/admin/stats
Deploy on Fly.io
Free tier · Always on · Needs terminal
0 of 4 done
1
Setup
Install Fly CLI
Open Terminal (Mac) or Command Prompt (Windows) and run one command.
Mac / Linux:
copycurl -L https://fly.io/install.sh | sh
Windows (PowerShell):
copypowershell -Command "iwr https://fly.io/install.ps1 -useb | iex"
2
Auth
Create Account & Login
Sign up for a free Fly.io account directly from the terminal.
copyfly auth signup
Already have an account? Run fly auth login instead.
3
Deploy
Launch Your App
Navigate to your kc-final folder and run these two commands.
copy# Go to your folder (update path to where you unzipped) cd ~/Downloads/kc-final # First time setup fly launch --name kilacombs --no-deploy # Deploy fly deploy
When fly launch asks questions, press Enter to accept the defaults. It detects Node automatically.
4
Launch
Open & Verify
Fly gives you a URL like kilacombs.fly.dev. Open it with one command.
copyfly open
✅ This opens your live site in the browser automatically.
